[Important Practical Notes]
- There is no custom assembly partitioning yet. A change in one runtime script can affect the whole default Assembly-CSharp compile.
- Runtime-generated objects are intentionally centralized through RuntimeSpriteFactory and RuntimeRoundObject.
- Core match logic is mostly concentrated in MatchSession. Scene code should prefer subscribing to GameManager events instead of reimplementing rules.
- ScoreChanger and TimerSystem currently look isolated from the main match architecture. Treat them as separate UI utilities unless they get wired back into GameManager/HUD flow.
- The direct dependency lines above were generated from symbol-reference scanning plus targeted code reading. Enum/type mentions are counted as dependencies even when usage is lightweight.
