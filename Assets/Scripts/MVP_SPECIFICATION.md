# GAMEJAM MVP仕様書

最終確認日: 2026-06-15
対象: Unity 6000.0.72f1 / `Assets/Scripts` 配下のランタイムコード / `Assets/GAMEJAM 概要書.txt`

## 1. この文書の位置付け

本書は、2人ローカル対戦ゲームとして最初に完成させるMVPの規範仕様を定義する。

- 「MVP仕様」は、完成判定に使う目標仕様である。
- 「現行実装」は、2026-06-15時点のコードから確認できる挙動である。
- 企画書とコードが食い違う場合、MVP完成時は本書のMVP仕様を優先する。
- 本書作成時点ではコード変更を行っていない。

## 2. MVP概要

### 2.1 ゲームの目的

2人のプレイヤーが、ゴールを目指す「ゴール役」と到達を阻止する「妨害役」に分かれて対戦する。両者が1回ずつゴール役を担当し、2ラウンドの結果で試合の勝者を決める。

### 2.2 プレイ構成

- プレイ人数: 2人固定
- 対戦形式: 1画面でのローカル対戦
- 入力機器: 2台のゲームパッドを基本要件とする
- マップ: 固定マップ1種類
- 試合ラウンド数: 2ラウンド固定
- 1ラウンドの進行: `Draw -> Place -> Race -> Result`

### 2.3 役割割り当て

| ラウンド | ゴール役 | 妨害役 |
|---|---|---|
| Round 1 | Player 1 | Player 2 |
| Round 2 | Player 2 | Player 1 |

Round 2のResult終了後に試合結果を表示し、Round 3には進まない。

## 3. ラウンド進行

### 3.1 フェーズ遷移

```text
Round Start
  -> Draw
  -> Place: ゴール役の設置3発
  -> Place: 妨害役の設置2発
  -> Race
  -> Result
  -> Round 2なら役割交替してRound Start
  -> 2ラウンド終了ならMatch Result
```

| フェーズ | 標準時間 | 終了条件 | 次フェーズ |
|---|---:|---|---|
| Draw | 20秒 | 時間切れ | Place |
| Place | 20秒 | 両者が規定弾数を撃ち終える、または時間切れ | Race |
| Race | 60秒 | ゴール役がゴール、または時間切れ | Result |
| Result | 6秒 | 時間切れ | 次ラウンドまたはMatch Result |

時間は `MatchConfiguration` の初期値に合わせる。調整可能な値とするが、全プレイヤーに同一条件を適用する。

### 3.2 Drawフェーズ

目的は、Placeフェーズで発射する図形を準備することである。

MVPでは自由曲線の手描きを必須とせず、以下の固定図形による簡易作成を採用する。

- 選択可能形状: Square / Circle / Triangle
- 両プレイヤーは同時に操作できる
- 移動入力で描画カーソルを動かす
- 図形切替入力で形状を変更する
- 攻撃入力で図形を配置する
- 描画カーソルは画面内の描画可能範囲から出られない
- キャラクター本体の移動、ジャンプ、ダッシュ、大砲操作は無効
- 配置数には残数制限を設ける

現行設定値に合わせ、ゴール役は3から6個、妨害役は2から5個をラウンド開始時にランダム配布する。各プレイヤーが配置したスタンプ群を1つの図形データとして確定し、その役がPlaceで撃つ全設置弾に同じ図形データを使用する。

### 3.3 Placeフェーズ

目的は、Raceフェーズで使う足場または障害物をマップ上へ設置することである。

- ゴール役が先攻し、3発を発射する
- ゴール役が3発を使い切ると妨害役へ操作権を移す
- 妨害役は後攻し、2発を発射する
- 妨害役が2発を使い切ると即座にRaceへ進む
- 発射回数はラウンドごとに `ゴール役3 / 妨害役2` へ戻す
- 4隅の大砲から1台を選択する
- 発射物は重力と回転を伴って飛ぶ
- 停止入力で発射物をその場に固定し、足場または障害物にする
- 発射済みの弾を停止してから次弾を撃つ
- Place中はキャラクターの移動、ジャンプ、ダッシュ、ゴール判定を無効にする
- 時間切れ時は未使用弾を失い、その時点の配置でRaceへ進む

企画書にあるパワーゲージと角度ゲージはMVP対象外とし、MVPでは4隅ごとの固定角度・固定初速を使用する。

### 3.4 Raceフェーズ

目的は、ゴール役が制限時間内にゴールゾーンへ到達することである。

ゴール役:

- 左右移動、ジャンプ、壁ジャンプ、ダッシュを使用できる
- ゴールゾーンへ入ると即座にラウンド成功となる
- Placeで固定した図形を足場として利用できる

妨害役:

- キャラクター本体は非表示、移動不可、当たり判定なしとする
- MVPではRace中の追加妨害弾を使用しない

共通制限:

- Race中はDraw操作と設置用砲撃を無効にする
- ゴール判定はRace中のゴール役だけを対象にする
- Race開始時からゴール到達までの経過時間を記録する
- ゴール方向への最大到達度を毎フレーム記録する

### 3.5 Resultフェーズ

- ゴール到達時はゴール役のラウンド勝利
- 時間切れ時は妨害役のラウンド勝利
- ラウンド勝者、ゴール成否、所要時間または最大到達度を表示する
- Result中は両プレイヤーの移動、描画、砲撃、ゴール判定を無効にする
- Round 1終了後は役割を交替してRound 2へ進む
- Round 2終了後は試合勝者を表示し、タイトルへ戻る操作と再戦操作を受け付ける

## 4. 操作仕様

### 4.1 ゲームパッド

| 入力 | Draw | Place | Race |
|---|---|---|---|
| 左スティック | 描画カーソル移動 | 使用しない | ゴール役の移動 |
| Southボタン | 使用しない | 使用しない | ジャンプ / 壁ジャンプ |
| Right Shoulder | 図形配置 | 選択中の大砲から発射 | 使用しない |
| Westボタン | 図形切替 | 使用しない | 使用しない |
| Eastボタン | 使用しない | 発射物を停止・固定 | 使用しない |
| D-pad Left / Right | 使用しない | 前 / 次の大砲を選択 | 使用しない |
| Left Shoulder | 使用しない | 使用しない | ダッシュ |

### 4.2 キーボード代替操作

キーボード操作は開発・検証用の代替手段であり、2ゲームパッド要件の代わりにはしない。

| 操作 | キー |
|---|---|
| 移動 / カーソル移動 | WASD または矢印キー |
| ジャンプ | Space |
| 図形配置 / 発射 | Enter |
| 図形切替 | X |
| 発射物停止 | C |
| 前 / 次の大砲 | 1 / 2 |
| ダッシュ | Left Shift |

## 5. 勝敗判定

### 5.1 ラウンド勝敗

- ゴール役がRace制限時間内にゴール: ゴール役に1ラウンドポイント
- Raceが時間切れ: 妨害役に1ラウンドポイント
- 1ラウンドで両者に同時加点しない

### 5.2 試合勝敗

2ラウンド終了時にラウンドポイントを比較し、多いプレイヤーを勝者とする。

### 5.3 1対1になった場合の判定

1対1になるのは、両プレイヤーがゴールに成功した場合、または両プレイヤーがゴールに失敗した場合である。各プレイヤーがゴール役だったラウンドの成績を次の順に比較する。

1. 両者ゴール成功なら、Race開始からゴールまでの所要時間が短いプレイヤーの勝利
2. 両者ゴール失敗なら、ゴール方向への最大到達度が高いプレイヤーの勝利
3. 所要時間または最大到達度が同一なら、試合は引き分け

最大到達度は、Race中にゴール役が到達した最大X座標を、スタートX座標からゴールX座標までの0.0から1.0に正規化して記録する。落下や後退によって値を減らさない。

## 6. 初期化仕様

### 6.1 試合開始時

- 現在ラウンドを1にする
- Player 1をゴール役、Player 2を妨害役にする
- 両プレイヤーのラウンドポイントを0にする
- 両プレイヤーのゴール所要時間を未記録にする
- 両プレイヤーの最大到達度を0にする
- ラウンド結果と試合結果を未確定にする
- Drawフェーズを開始し、Drawタイマーを設定する

### 6.2 ラウンド開始時

- フェーズ結果を `None` にする
- Draw用図形残数を役割ごとに再配布する
- Place用発射残数を `ゴール役3 / 妨害役2` にする
- 現在のPlace操作権をゴール役にする
- Race経過時間と最大到達度を0にする
- 両プレイヤーを役割別スポーン地点へ戻す
- 移動入力、速度、角速度、入力ロック、スタンを解除する
- ダッシュの継続時間、クールダウン、使用可否を初期化する
- 選択中の大砲を先頭へ戻す
- 選択中の図形をSquareへ戻す
- 描画カーソル位置を初期位置へ戻す
- 前ラウンドの配置図形、飛翔中の弾、停止済みの弾、選択マーカーを削除する
- プレイヤーの表示、コライダー、移動ロックを現在フェーズと役割に合わせる

### 6.3 ラウンド終了時

- ラウンド勝者へ1ポイント加算する
- ゴール成否を保存する
- 成功時はRace所要時間を保存する
- 失敗時は最大到達度を保存する
- 以降のゴール判定とスコア加算を停止する
- Resultフェーズの操作ロックを適用する

### 6.4 タイトルへ戻る時 / 再戦時

- ラウンド番号、フェーズ、役割、結果、タイマーを初期値へ戻す
- 両プレイヤーのポイント、所要時間、最大到達度を消去する
- 発射残数と図形残数を0にする
- ランタイム生成物を削除する
- 再戦時は「試合開始時」の処理から開始する

## 7. MVP対象外

- 自由曲線による本格的なお絵描き
- 色選択、ブラシ種類、消しゴム、Undo / Redo
- 描いた絵へのランダム特殊効果
- Race中に妨害役が撃つ追加スタン弾
- パワーゲージ、角度ゲージ、任意照準
- 爆弾、凍結爆弾などのアイテム選択
- 複数マップ、ランダムマップ、ステージ選択
- 3人以上の対戦
- オンライン対戦
- AI対戦
- キャラクター選択、スキン、成長要素
- Steam / Nintendo Switch固有機能、実績、ランキング
- 詳細チュートリアル、演出強化、サウンドの完成版

## 8. 現行コードの実装状況

判定:

- 実装済み: MVP仕様を満たす主要コードがある
- 部分実装: コードはあるが、制限・接続・完了条件に不足がある
- 未実装: 対応する状態または処理がない

| 項目 | 状況 | 現行実装と差分 |
|---|---|---|
| Draw -> Place -> Race -> Result | 実装済み | `MatchSession.AdvancePhase` で時間進行する。Placeは弾を使い切ってもRaceへ進む。 |
| フェーズ時間 | 実装済み | 初期値はDraw 20秒、Place 20秒、Race 60秒、Result 6秒。 |
| ゴール役3発 / 妨害役2発 | 実装済み | Place開始時に残弾を設定し、ゴール役から妨害役へ操作権を渡す。 |
| 2人ローカル生成 | 部分実装 | Remaシーンの1人目を複製して2人目を作る。Gameシーンにはプレイヤーがなく、実プレイ導線はTitleからRema。 |
| 2台ゲームパッドの独立割当 | 部分実装 | `Gamepad.current` を使うため、2台を別プレイヤーへ確実にペアリングする処理がない。 |
| 役割交替 | 部分実装 | ラウンド更新時にプレイヤーの役割は反転するが、ラウンド数に上限がなく毎ラウンド反転し続ける。 |

## 9. 企画書との主な差分

### 9.1 MVPとして意図的に縮小するもの

- 自由手描きを固定3図形へ簡略化する
- パワーと角度の2ゲージを固定角度・固定初速へ簡略化する
- Race中の妨害弾をMVPから外す
- アイテム、ランダム特殊効果、完成版チュートリアルを外す

### 9.2 現行コードが企画書より先行しているもの

- 固定図形スタンプと個数予算
- ランタイムHUD
- 妨害用スタン弾
- ラウンド生成物をまとめて破棄する仕組み

### 9.3 現行コードが企画書またはMVP仕様を満たしていないもの

- 2ラウンドで試合を終了し、各プレイヤーのゴール役を1回に限定する処理
- プレイヤー別の得点、ゴール時間、最大到達度、同点判定
- 2台のゲームパッドを別々のプレイヤーへ割り当てる処理
- Drawで作った内容をPlaceの発射物として使用する接続
- Place / Result中のキャラクター操作禁止
- Race中かつゴール役だけに限定したゴール判定
- ラウンドごとのDraw残数再配布
- 最終結果画面と再戦導線

## 10. MVP完成条件

次の項目をすべて満たした時点をMVP完成とする。

1. 2台のゲームパッドで2人が独立操作できる
2. 1試合が2ラウンドで終了する
3. 両プレイヤーがゴール役を1回ずつ担当する
4. 全ラウンドが `Draw -> Place -> Race -> Result` の順で進む
5. Placeの発射数がゴール役3発、妨害役2発に制限される
6. Race中のゴールまたは時間切れでラウンド勝者が1人に確定する
7. 2ラウンドのポイントと同点判定から試合結果が確定する
8. 各フェーズで許可されていない操作が無効になる
9. ラウンド開始時に残数、盤面、プレイヤー状態、計測値が正しく初期化される
10. Round 2終了後にRound 3へ進まず、最終結果と再戦 / タイトル操作を表示する
